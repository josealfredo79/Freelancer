🤔 1. ¿Qué es un Contrato Escrow?

💰 Definición Simple

Un escrow es como un triángulo de confianza. El dinero pasa a un "contenedor inteligente" y solo se libera cuando el trabajo está hecho y aprobado. Ninguna de las partes puede tomarse el dinero por su cuenta.

1

El Problema que Resuelve

En trabajos freelance tradicionales hay dos problemas:

  • 😰 La empresa paga y el freelancer no entrega
  • 😱 El freelancer entrega y la empresa no paga
✅ Solución: El contrato inteligente automatiza todo. El dinero queda bloqueado hasta que ambas partes cumplen su palabra.
2

¿Por qué en Stellar?

Stellar es ultra-rápido (3-5 segundos), ultra-barato ($0.0001 por transacción) y muy seguro. Además, Soroban permite lógica personalizada con contratos inteligentes.

👥 2. Los 3 Actores del Contrato

ℹ️ Importante: Todo contrato escrow tiene 3 participantes claramente definidos desde el inicio.
🏢

Empresa

Es quien paga y define las condiciones del trabajo.

  • Inicializa el contrato
  • Deposita los fondos
  • Aprueba hitos completados
  • Autoriza pagos
  • Puede iniciar disputa
👨‍💻

Freelancer

Es quien hace el trabajo y recibe el pago.

  • Trabaja en los hitos
  • Entrega el trabajo
  • Recibe pagos por hito
  • Puede iniciar disputa
  • No puede拿走 fondos
⚖️

Árbitro

El mediator neutral que resuelve conflictos.

  • Recibe confianza de ambos
  • Solo interviene en disputas
  • Decide distribución de fondos
  • Resuelve conflictos
  • NO tiene acceso al dinero por defecto

Estado Descripción ¿Qué puede pasar?
Activo Contrato creado, sin fondos aún La empresa puede depositar o cancelar
Depositado Dinero bloqueado en el contrato Aprobación + Release, Disputa, o Cancela
Completado Todos los hitos pagados Contrato terminado (final)
Disputado Conflicto entre partes Solo el árbitro puede resolver
Cancelado Contrato terminado Fondos devueltos (final)

🎯 5. Sistema de Hitos (Milestones)

Los hitos dividen el proyecto en fases pequeñas para pagos graduales:

Hito 1 $500 USD
Diseño UI/UX inicial y mockups
✓ Completado y Pagado
Hito 2 $1,500 USD
Desarrollo del frontend
✓ Completado y Pagado
Hito 3 $2,000 USD
Backend e integración
⏳ En Progreso
Hito 4 $1,000 USD
Testing y deployment
⏳ Pendiente
⚠️ Regla Importante: Para aprobar un hito y liberar el pago: (1) el Freelancer marca como completado, (2) la Empresa verifica y APRUEBA manualmente. Ambos deben estar de acuerdo.

⚙️ 6. Funciones del Contrato (API)

Estas son todas las operaciones que puedes ejecutar:

Función Quién la llama Qué hace
initialize 🏢 Empresa Crea el contrato con las direcciones de los 3 participantes y la lista de hitos
deposit 🏢 Empresa Deposita el monto total (todos los hitos) al contrato
approve_milestone 🏢 Empresa Aprueba que un hito está completado y listo para pagar
release 🏢 Empresa Libera el pago del hito aprobado al freelancer
dispute 🏢 o 👨‍💻 Abre una disputa si hay desacuerdo
resolve ⚖️ Árbitro Resuelve la disputadecidiendo cuántos fondos van a cada parte
cancel 🏢 Empresa Cancela el contrato (si no hay hitos aprobados, devuelve fondos)
query_escrow Cualquiera Lee el estado actual del contrato (sin costo)
🚨 Errores Comunes a Evitar:

1. El doble guion (--): Al invocar funciones en CLI, usa -- antes del nombre de la función.
2. Números i128: Siempre pasa los números entre comillas: "500000000" (no 500000000).
3. Autenticación: Cada función verifica que el que llama tenga autorización.

🚀 12. Guía de Despliegue a Testnet

Sigue estos pasos para compilar, configurar y desplegar tu contrato inteligente en la testnet de Stellar. Cada comando está listo para copiar y pegar.

1

Requisitos Previos

🦀
Rust
rustup.rs
⚡
Stellar CLI
stellar-cli
🐳
Docker
Optional
2

Instalar Herramientas

💡 Primero: Si no tienes Rust, instálalo. Stellar CLI te permitirá compilar y desplegar contratos. 
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
cargo install stellar-cli --locked
3

Configurar Wallet

⚠️ Importante: Crea una clave para la testnet. Usa el mismo nombre en todos los comandos. 
stellar keys generate freelancer
stellar keys address freelancer

Resultado: Te mostrará una dirección como GBSA...

4

Fondear Cuenta (Testnet)

💡 Friendbot: Ve a friendbot.stellar.org y pega tu dirección para obtener XLM de prueba gratis.

O usa este comando para fondeo automático:

stellar keys fund freelancer --network testnet
5

Compilar a WebAssembly

⚠️ Debes usar el target wasm32: Stellar usa WebAssembly para ejecutar contratos de forma segura y rápida. 
cargo build --release --target wasm32-unknown-unknown

Esto generará el archivo:

target/wasm32-unknown-unknown/release/freelancer_escrow.wasm

6

Optimizar el WASM

⚠️ Importante: Siempre optimiza antes de desplegar. Esto elimina código no necesario y reduce errores. 
stellar contract optimize --wasm target/wasm32-unknown-unknown/release/freelancer_escrow.wasm

Esto creará:

freelancer_escrow.optimized.wasm

7

Subir a Testnet

✅ Este comando hace 2 cosas: (1) Sube el WASM a la red, (2) Obtiene el hash para deploy. 
WASM_HASH=$(stellar contract upload --network testnet --source freelancer --wasm target/wasm32-unknown-unknown/release/freelancer_escrow.optimized.wasm) && echo "WASM Hash: $WASM_HASH"
8

Desplegar el Contrato

✅ Último paso: Despliega el contrato y guarda el ID del contrato. 
CONTRACT_ID=$(stellar contract deploy --network testnet --source freelancer --wasm-hash $WASM_HASH) && echo "Contract ID: $CONTRACT_ID"
💡 También puedes usar el contrato ya desplegado: CAVOGXUTNJ4OVDT2H2GVCP6DOFZCF6CLKFIKWAJGYKI2MAZM3EUC4SEM
9

Inicializar el Contrato

Antes de usar, debes inicializar con las direcciones de los participantes y los hitos:

stellar contract invoke \
  --id CAVOGXUTNJ4OVDT2H2GVCP6DOFZCF6CLKFIKWAJGYKI2MAZM3EUC4SEM \
  --source freelancer \
  --network testnet \
  -- \
  initialize \
  --empresa GBSA27XJEXNH4N276LHG2YJXR7RMKWCBYB7ICUSBNQSFMKORFDDY4Q5C \
  --freelancer GBSA27XJEXNH4N276LHG2YJXR7RMKWCBYB7ICUSBNQSFMKORFDDY4Q5C \
  --arbitro GBSA27XJEXNH4N276LHG2YJXR7RMKWCBYB7ICUSBNQSFMKORFDDY4Q5C \
  --token CDLZFC3T3U2CQTJQDXB2D4M5Z6N7P8Q9R0S \
  --hitos '[{"id": 1, "descripcion": "Fase 1", "monto": "500000000", "completado": false, "aprobado": false}]'
✅ Este comando hace 2 cosas: (1) Sube el WASM a la red, (2) Obtiene el hash para deploy. 
WASM_HASH=$(stellar contract upload --network testnet --source freelancer --wasm target/wasm32-unknown-unknown/release/freelancer_escrow.optimized.wasm) && echo "WASM Hash: $WASM_HASH"
8

Desplegar el Contrato

✅ Último paso: Despliega el contrato y guarda el ID del contrato.

Inicializar el Contrato

Antes de usar, debes inicializar con las direcciones de los participantes y los hitos:
