🎯 Objetivos de Aprendizaje

Al terminar este tutorial podrás:

  • Instalar y configurar el entorno de desarrollo para Stellar Soroban
  • Compilar un contrato inteligente de Rust a WebAssembly (WASM)
  • Desplegar el contrato en la testnet de Stellar
  • Interactuar con el contrato mediante invoked transactions
¿Qué es un contrato escrow? Un contrato escrow es un acuerdo neutral donde un tercero retiene el dinero hasta que se cumplan ciertas condiciones. En este caso, el contrato inteligente de Stellar retiene los fondos hasta que el cliente aprueba cada "hito" (milestone) completado por el freelancer.

⚙️ Instalación del Entorno de Desarrollo

Antes de escribir nuestro contrato inteligente, necesitamos configurar las herramientas de desarrollo. Esto incluye el lenguaje de programación Rust y la CLI de Stellar.

1Instalar Rust

¿Por qué Rust? Stellar Soroban utiliza Rust porque es un lenguaje moderno, seguro y eficiente que compila a WebAssembly. WebAssembly (WASM) es un formato de código binario que se ejecuta en cualquier navegador o blockchain de manera segura y rápida. 
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
💡 Consejo: Durante la instalación,选择选项 1 (default) para instalar todas las herramientas necesarias.

2Verificar instalación de Rust

rustc --version && cargo --version

3Instalar Stellar CLI

¿Qué es la Stellar CLI? Es una herramienta de línea de comandos que nos permite compilar, desplegar e interactuar con contratos inteligentes en la red Stellar. 
cargo install stellar-cli --locked
⚠️ Nota: La compilación puede tomar varios minutos (5-15) dependiendo de tu computadora.

4Verificar Stellar CLI

stellar --version

🔨 Compilar el Contrato Inteligente

Ahora vamos a configurar nuestro proyecto para que compile a WebAssembly. WebAssembly es un formato de código binario que se ejecuta en la blockchain de Stellar.

1Agregar crate-type al Cargo.toml

¿Por qué necesitamos esto? Necesitamos generar dos tipos de archivos:
  • cdylib: Biblioteca dinámica en C para que pueda ser llamada desde fuera (el contrato desplegable)
  • rlib: Biblioteca estática de Rust para testing interno

Abre el archivo Cargo.toml de tu proyecto y agrega:

[lib]
crate-type = ["cdylib", "rlib"]

2Instalar objetivo WebAssembly

¿Qué es wasm32-unknown-unknown? Es la arquitectura objetivo para compilar Rust a WebAssembly. Significa:
  • wasm32: Arquitectura de 32 bits
  • unknown: Sistema operativo desconocido (WebAssembly no tiene SO)
  • unknown: Target de entorno desconocido
rustup target add wasm32-unknown-unknown

3Compilar a WebAssembly

cargo build --release --target wasm32-unknown-unknown

4Verificar archivo generado

ls -la target/wasm32-unknown-unknown/release/freelancer_escrow.wasm
💡 Consejo: El archivo .wasm debería tener un tamaño entre 20KB y 50KB aproximadamente.

🚀 Desplegar el Contrato en Testnet

¿Por qué Testnet? La testnet es una red de pruebas de Stellar donde no usamos dinero real. Es el entorno ideal para aprender y experimentar antes de usar la red principal (Mainnet).

1Generar par de claves (密钥对)

¿Qué es una clave pública y privada?
  • Dirección pública (Public Key): Como tu número de cuenta bancaria, la comparten para recibir pagos
  • Clave privada (Secret Key): Como tu contraseña, NUNCA la compartas con nadie
stellar keys generate testuser

2Obtener XLM de prueba

¿Qué es Friendbot? Es un servicio que otorga XLM de prueba gratuitos para desarrolladores. Solo funciona en testnet, no tiene valor real. 
curl "https://friendbot.stellar.org/?addr=TU_DIRECCION_PUBLICA"
⚠️ IMPORTANTE: Copia tu dirección pública del paso anterior y reemplaza TU_DIRECCION_PUBLICA en el comando.

3Desplegar el contrato

stellar contract deploy \
  --wasm target/wasm32-unknown-unknown/release/freelancer_escrow.wasm \
  --source testuser \
  --network testnet
💡 Recuerda: Copia el CONTRACT_ID que te devuelve este comando. Lo necesitarás para invocar el contrato.

📞 Invocar el Contrato Inteligente

¿Qué es una invocación? Llamar a una función del contrato inteligente en la blockchain. Cada invocación es una transacción que se registra en la blockchain.
⚠️ IMPORTANTE: En todos los comandos siguientes, reemplaza:
  • CONTRACT_ID con el ID de tu contrato (del paso anterior)
  • ADDR_EMPRESA con la dirección de la empresa (tu cuenta testuser)
  • ADDR_FREELANCER con otra cuenta que generes
  • ADDR_ARBITRO con otra cuenta adicional

1Crear cuentas adicionales

stellar keys generate freelancer
stellar keys generate arbitro

2Inicializar el contrato

¿Qué hace initialize? Configura el contrato con las direcciones de las partes y los hitos definidos. Solo se ejecuta una vez al principio. 
stellar contract invoke \
  --id CONTRACT_ID \
  --source testuser \
  --network testnet \
  initialize \
  --empresa "ADDR_EMPRESA" \
  --freelancer "ADDR_FREELANCER" \
  --arbitro "ADDR_ARBITRO" \
  --token "ADDR_TOKEN" \
  --hitos '[{"id":1,"descripcion":"Desarrollo demo","monto":1000}]'

3Depositar fondos

¿Qué hace deposit? La empresa deposita el monto total de los hitos. El contrato retiene estos fondos hasta que se cumpla cada hito. 
stellar contract invoke \
  --id CONTRACT_ID \
  --source testuser \
  --network testnet \
  deposit \
  --monto 1000

4Aprobar hito

¿Qué hace approve_milestone? La empresa marca un hito como completado. Esto prepara los fondos para ser liberados. 
stellar contract invoke \
  --id CONTRACT_ID \
  --source testuser \
  --network testnet \
  approve_milestone \
  --hito_id 1

5Liberar pago

¿Qué hace release? Transfiere los fondos del hito al freelancer. Solo se puede ejecutar después de aprobar el hito. 
stellar contract invoke \
  --id CONTRACT_ID \
  --source testuser \
  --network testnet \
  release \
  --hito_id 1

6Consultar estado

¿Qué hace query_escrow? Retorna el estado actual del contrato: partes, hitos completados, fondos disponibles, etc. 
stellar contract invoke \
  --id CONTRACT_ID \
  --source testuser \
  --network testnet \
  query_escrow

📋 Resumen de Funciones del Contrato

FunciónDescripciónQuién puede ejecutarlo
initializeInicializar contrato con partes e hitosEmpresa
depositDepositar monto total de hitosEmpresa
approve_milestoneMarcar hito como completadoEmpresa
releaseLiberar pago al freelancerEmpresa
disputeAbrir disputaEmpresa o Freelancer
resolveResolver disputaÁrbitro
cancelCancelar y devolver fondosCualquiera
query_escrowConsultar estado actualCualquiera
